Michał Pastuszko Lab02 sprawozdanie

1. Wygenerowano klucz SSH poleceniem ssh-keygen -t ed25519 -C, utworzono do niego hasło, dodano do agenta ssh poleceniem ssh-add, następnie skopiowano klucz publiczny i zarejestrowano go po stronie serwera. Ustalono, że zmiana sposobu logowania powiodła się.
2. Dodano tag do aktualnego commita poleceniem git tag -a MP306484_Lab02 -m. Upewniono się że tag został dodany poleceniem git show MP306484_Lab02. Wysłano dokonaną zmianę na zdalne repozytorium polecieniem git push --follow-tags -set-upstream origin MP306484
3. Stworzono(za pomocą eksploratora Windows, nie konsoli) folder Grupy, w nim podfolder Grupa04. Za pomocą polecenia git mv MP30484/ Grupy/Grupa04/ przeniesiono mój folder do nowej lokacji. Następnie stworzono nowy branch krótkożyjący Grupa04-MP306484_Lab02 i przełączono się na niego poleceniem git checkout -b Grupa04-MP306484_Lab02
4. W folderze .git/hooks zmodyfikowano plik commit-msg w taki sposób, aby każdy commit message musiał się zaczynać od MP306484. W tym celu napisano skrypt przedstawiony na screenshocie zadanie4.1. Usunięto rozszerzenie sample i przetestowano działanie hooka, co przedstawiono na screenshotach zadanie4.2 i zadanie4.3. Hook działa prawidłowo, ponieważ dopuszcza jedynie commity z messagem zaczynającym się od MP306484, nie dopuszcza takich które nie zawierają tej frazy, ani takich które frazę zawierają, ale nie na początku.
5. Przełączono się na branch grupowy za pomocą git checkout Grupa04, z jego poziomu zmergowano do niego mój osobisty branch używając git merge Grupa04-MP306484_Lab02, zmiany spushowano na zdalne repozytorium poleceniem git push. Następnie usunięto branch osobisty poleceniem git branch -D Grupa04-MP306484_Lab02.
